1、内部类
定义：内部类就是定义在另一个类的内部的类，与之对应的类叫做外部类，换种说法就是一个类的定义
出现在另一个类的内部，并将该类称为“寄生类”。

为什么要有内部类：每一个内部类都可以独自的去继承一个类或者实现若干个接口，外部类是否继承有
父类，对于内部类没有影响，从这个角度来看的话，内部类使得多继承加以实现了。接口解决了部分的问题，
而内部类有效的实现了“多重继承”。

class 教室  extends 建筑物{
	class  课桌 extends XXX；
	class  学生 extedns XXX;	
}

内部类分为四种：
1)成员内部类(实例内部类)
2)局部内部类(方法内部类)
3)静态内部类
4)匿名内部类

成员内部类(实例内部类)：
    定义在外部类的内部，相当于外部类的一个属性，可以使用任意类型的访问控制修饰符。
    
    定义了成员内部类之后，必须使用外部类对象来创建内部类对象，内部类的对象必须依赖于外部类
的对象而存在，外部类及其对象不能直接访问内部类的方法和属性，可以先创建内部类对象，然后通过此对象来访问其方法和属性。
    语法：内部类型 变量名 = 外部类对象.new 内部类型()
    
    当内部类的属性和方法与外部类的属性和方法重名时，内部类可以通过外部类名.this.属性的方式
来访问外部类的属性或方法。
